# Android 앱 안내

이 폴더는 Android 클라이언트 구현을 위한 기준 문서를 담는 곳이에요.

아직 실제 앱 코드는 없고, 서버를 먼저 개발하는 단계예요. 그래도 Android 쪽 구현이 시작될 때 혼선이 없도록 역할과 제약을 먼저 문서로 고정해둘게요.

## 이 영역의 역할

Android 앱은 아래 책임을 맡아야 해요.
- Mac과의 페어링을 시작하거나 완료해야 해요.
- Android 알림을 수집해서 암호화한 뒤 relay 서버로 보내야 해요.
- Android 클립보드를 읽을 수 있는 상황에서만 안전하게 수집해야 해요.
- Mac에서 온 클립보드 payload를 Android 클립보드에 반영해야 해요.
- 키와 relay 인증 정보를 안전한 저장소에 보관해야 해요.

## 현재 상태

- 구현 상태: 아직 프로젝트 생성 전이에요.
- 우선순위: relay 서버 다음이에요.
- 제약 사항: Android 10 이상에서는 일반 앱의 백그라운드 클립보드 읽기가 막혀 있어요.

## Android에서 꼭 기억해야 할 결정

- 백그라운드 상시 클립보드 감시는 v1 범위에서 불가능해요.
- 따라서 `Android -> Mac` 방향은 전경 감시 또는 수동 전송이어야 해요.
- Android 알림은 `NotificationListenerService`로 수집해야 해요.
- 지원 클립보드 포맷은 공통 교집합 포맷만 다뤄야 해요.
- payload는 기기에서 암호화한 뒤 relay로 보내야 해요.
